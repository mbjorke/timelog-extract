# `/gittan-product-owner` (gittan / planning)

Thin wrapper. The canonical workflow lives in
**`docs/skills/gittan-product-owner.md`** — read and follow it.

**Use when:** prioritizing, shaping a backlog, slicing a feature, deciding "X
before Y?", or writing acceptance criteria. This is a **planning** pass: produce
ordered backlog items + Gherkin where useful, **not** code.

- Attach the relevant docs/specs with `@`; start from
  `docs/product/vision-documents.md` for product framing.
- Output backlog items in the shape defined in the canonical doc; mark each
  `now` / `next` / `later` / `do not build yet`.
