## 2026-07-02

- **Two agents in one clone will collide silently — recover via reflog + a safety tag.** Claude Code and Cursor both worked `task/*` branches in the same working tree; a reused branch name + a checkout/reset left one agent's committed work dangling (no branch pointed at it). It was recovered from `git reflog` and pinned with `git tag safety/<name> <sha>`. Guardrails for next time: give each lane a **distinct, clearly-owned branch name**, work in your **own `git worktree`**, `git tag safety/...` before stepping away, and check local lanes/worktrees **before** branching (what `scripts/rabbit_workflow_context.sh` now automates — GH #240).
- **A fresh worktree is not an isolated dev environment.** `scripts/run_lint.sh` (run by `run_autotests.sh`) looks for `<root>/.venv/bin/ruff`; a new worktree has no `.venv`, so the lint gate exits non-zero under `set -e` and the unit tests never run — which misreads as a **test failure**. Fix: symlink the main clone's env (`ln -sfn <main-clone>/.venv .venv`) and add `.venv` to `.git/info/exclude`, or `pip install -e '.[dev]'` in the worktree.
- **Shared git identity means commits aren't agent-attributable.** Every agent commits as the same `user.name`/`user.email`, so history (and reflog) shows *what* changed, not *who* did it. Confirm "who created branch X" from the **branch reflog** (`branch: Created from …`), not authorship. The durable fix — an `Agent:` commit trailer + a branch-claim registry the loop checks — is tracked in GH #272 (complementary to worktree isolation in #240).
