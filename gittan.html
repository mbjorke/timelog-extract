The problem

AI work is fragmented.
Memory is not enough.

In one 15-minute window you can touch several agents, several repos, and several clients. The hard part is no longer proving that work happened. It is explaining which work belonged where.

Agent chaos

Many tools, many threads

Cursor, Claude Code, GitHub, browser research, docs, and email all leave partial traces of the same workday.

Config friction

Useful mapping takes effort

The current beta blocker is time-to-useful-project-config: turning raw traces into project rules a person can trust.

Truth split

Classified is not approved

Gittan separates observed traces, classified candidates, and human-approved invoice time.

Local-first

Your raw day stays local

Gittan is built around local processing by default, because work traces are personal and often client-sensitive.

Current product shape

Local traces become
reviewable evidence

Gittan is a macOS-first CLI today. It can generate local reports, source summaries, and exports. The next milestone is making the first useful project config much easier.

Run locally

Install the CLI and keep raw work traces on your machine by default:

pipx install timelog-extract

Python 3.9+. Use gittan doctor to inspect setup and source availability.

Generate evidence

Start with a normal report and source summary:

gittan report --today --source-summary

Improve mapping

The beta onboarding path uses read-only triage before any config write:

gittan triage --json
Timelog Truth Standard

Observed. Classified.
Approved by a human.

The product direction is simple: do not confuse evidence with approval. Gittan helps you inspect the chain before anything becomes invoice-relevant.

Observed time

What the machine can see

Local traces from IDEs, agents, browser history, worklogs, commits, and GitHub activity.

Classified time

What the rules suggest

Project mappings are candidates with reasons, not magic truth. Broad rules and uncertainty should stay visible.

Approved time

What a human accepts

Invoice-relevant time should come from explicit approval, not a black-box classifier.

Evidence plan

Config onboarding path

Read-only triage shows top domains, timestamp hints, and suggested mappings before any config write.

Local-first

No raw-day cloud lake

The default posture is local processing and explicit export, because work history is sensitive.

Privacy-first architecture

Your work history
stays on your machine

Raw activity data never leaves your computer. Not for analytics. Not for features. Not for anything.

  • All signal collection runs locally — Chrome history, IDE logs, AI sessions
  • No account required. No telemetry. No usage tracking
  • Optional GitHub integration uses your own token against GitHub's public API
  • Invoice PDFs generated on your machine — no SaaS PDF service involved
  • GPL-3.0-or-later licensed — read every line if you want to

Made in Åland

Built on an island
in the Baltic Sea

Gittan is made in Åland — an autonomous archipelago between Sweden and Finland with its own parliament (Lagting), its own legislation, its own flag, and its own postage stamps. Not a country, but not quite a region either. Something more interesting than both.

If you've ever considered working remotely somewhere that isn't a co-working space in a major city, Åland is worth a serious look. 30,654 inhabitants across 6,700 islands, clean air, and a growing number of people who build things from here.
