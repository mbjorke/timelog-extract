# Inspiration (links, patterns, external sparks)

**Not** the same as [`docs/ideas/`](../ideas/) (product bets and working strategy) or [`docs/specs/`](../specs/) (build contracts).

Use this folder for **short** notes that help contributors and agents **orient faster**:

- Links to articles, repos, or talks that influenced a decision (with one-line “why it matters”).
- UX or CLI patterns (“like tool X does Y”) before they become a spec.
- Reminders of ecosystem moves (new editor, new agent product) — **not** marketing copy.

## Rules

- Prefer **English** in committed files (same as `CONTRIBUTING.md`).
- If an inspiration becomes **policy** or **shipped behavior**, promote it to **`docs/decisions/`** or **`docs/specs/`** and leave a pointer here or delete the stub.
- Do not dump **private** or **user-identifying** content; see `AGENTS.md` → documentation privacy.

## Naming

- Kebab-case files, e.g. `cli-patterns-from-other-tools.md`, `2026-agent-ecosystem-notes.md`.
